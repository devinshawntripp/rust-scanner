   ```rust
   // AFTER:

   ```

2. In `epss_enrich_findings`, add cluster-mode PG lookup BEFORE the existing file-cache/API path:
   - Collect all unique CVE IDs from findings
   - If `cluster_mode()` and `pg.is_some()`:
     - Call `pg_get_epss_scores(c, &cve_ids, compute_jittered_ttl_days(30, 7))`
     - Apply PG-cached scores to findings immediately
     - Remove PG-hit CVEs from the list to fetch
   - If NOT cluster_mode(): use existing file-cache path unchanged
   - After API fetch completes for any misses:
     - If `cluster_mode()` and `pg.is_some()`: call `pg_put_epss_scores(c, &fetched_scores)`
     - Else: write to file cache (existing behavior)

3. Change `kev_enrich_findings` signature in `src/vuln/kev.rs`:
   ```rust
   // BEFORE:
   pub fn kev_enrich_findings(findings: &mut [Finding], cache_dir: Option<&std::path::Path>)
   // AFTER:
   pub fn kev_enrich_findings(findings: &mut [Finding], pg: &mut Option<postgres::Client>, cache_dir: Option<&std::path::Path>)
   ```

4. In `kev_enrich_findings`, add cluster-mode PG lookup:
   - If `cluster_mode()` and `pg.is_some()`:
     - Call `pg_get_kev_entries(c, compute_jittered_ttl_days(30, 7))`
     - If result is non-empty, use it as the KEV set and skip API call
   - If NOT cluster_mode(): use existing file-cache path unchanged
   - After API fetch for cache miss:
     - If `cluster_mode()` and `pg.is_some()`: call `pg_put_kev_entries(c, &kev_ids)`
     - Else: write to file cache (existing behavior)

IMPORTANT: PgClient is !Send — do NOT try to use pg inside rayon parallel iterators. Keep PG operations sequential, same pattern as osv/enrich.rs and nvd/enrich.rs.
