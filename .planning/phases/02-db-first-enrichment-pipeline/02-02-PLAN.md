  <files>src/vuln/osv/batch.rs, src/vuln/osv/mod.rs</files>
  <action>
1. Change the signature of `osv_batch_query` in `src/vuln/osv/batch.rs`:
   ```rust
   // BEFORE:
   pub fn osv_batch_query(packages: &Vec<PackageCoordinate>) -> serde_json::Value {
   // AFTER:
   pub fn osv_batch_query(packages: &Vec<PackageCoordinate>, pg: &mut Option<postgres::Client>) -> serde_json::Value {
   ```

2. Inside the per-chunk loop, BEFORE the existing file-cache check (the `cache_get` call), add a cluster-mode PG check:
   ```rust
   // Cluster mode: check PG chunk cache first
   if crate::vuln::cluster_mode() {
       if let Some(c) = pg.as_mut() {
           let ttl = crate::vuln::pg::compute_jittered_ttl_days(30, 7);
           if let Some(cached) = crate::vuln::pg::pg_get_osv_batch_chunk(c, &body_digest, ttl) {
               if let Some(arr) = cached["results"].as_array() {
                   for (idx_in_chunk, item) in arr.iter().enumerate() {
                       if idx_in_chunk < chunk.len() {
                           results[chunk[idx_in_chunk].0] = item.clone();
                       }
                   }
               }
               crate::utils::progress("osv.query.chunk.pg_cache", &format!("digest={}", &body_digest[..8.min(body_digest.len())]));
               continue; // skip to next chunk
           }
       }
   }
   ```

3. In standalone mode, the existing file-cache path continues to work unchanged. In cluster mode, skip the file-cache check (add `if !crate::vuln::cluster_mode()` guard around the existing `cache_get` block).

4. After a successful API fetch and parse, add PG write-back for cluster mode:
   ```rust
   // Write back to appropriate cache
   if crate::vuln::cluster_mode() {
       if let Some(c) = pg.as_mut() {
           crate::vuln::pg::pg_put_osv_batch_chunk(c, &body_digest, &v);
       }
   } else {
       cache_put(cache_dir.as_deref(), &cache_tag, v.to_string().as_bytes());
   }
   ```
   Replace the existing unconditional `cache_put` call with this conditional block.

5. Update the re-export in `src/vuln/osv/mod.rs` — the `pub use batch::osv_batch_query;` line stays the same (signature change propagates automatically).

6. Do NOT change the per-package fallback loop (lines ~204-263) — that already has proper retry bounds. The circuit breaker integration for that loop happens in Plan 04.
